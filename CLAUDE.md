## 委譲ルール：別チャットへの依頼は「1コピペ依頼文」で出す

ユーザーが、ある作業を **別のチャット／別の LLM**（Codex、別の Claude
セッション、ローカル LLM 等）に投げようとしている時——
たとえば「Codex に投げる」「別チャットに頼む」「委譲したい」「依頼文作って」
といった意図が見えたら——

**ユーザーは要点だけ伝える。依頼文の完成形は Claude が作る。**

これは毎回やる。聞かれなくてもやる。脳死でコピペできる状態で渡す。

### 出力の形（厳守）

1. **単一のコードブロックに、完成した依頼文を丸ごと入れる。**
   ユーザーは中身を編集せず、そのままコピペするだけで済むこと（脳死コピペ）。

2. **プレースホルダを残さない。**
   `<対象パス>` のような空欄を作らない。会話から分かる情報は全て埋める。
   本当に不明な点だけ、コードブロックの**外**で「ここだけ教えて」と
   最小限に確認する（理想は確認ゼロ）。

3. **自己完結させる。**
   受け取る側のチャットは、このリポジトリもテンプレートも持っていない前提。
   必要な制約・原則は依頼文の中に直接書き下す。
   「第5条」等の参照だけで済ませず、一言で意味も添える。

4. **構造は `SEN_Delegation_Template_v1.md` の共通層に従う。**
   目的 / 文脈 / 成果物 / 制約（データ主権・摩擦配置・抽象不可侵）/
   評価基準 / 禁止事項 / 引き継ぎ。

5. **モデル非依存に書く（第11.3 共通層）。**
   特定モデルのツール名・口調を依頼文本体に埋め込まない。
   モデル固有の調整が要るなら、コードブロックの**後**に
   「アダプタ補足」として短く別記する。

### 原則

人間は抽象（何を・なぜ・どこまで）を渡す。
モデルは具体（どう）を実装する（第7条・第17条）。
依頼文は、その翻訳の器である（第5章）。
モデルは借り物（第25条）。器は手放さない。
